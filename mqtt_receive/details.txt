Ce dossier regroupe les programmes de la récéption des données du routeur via la MQTT. 
Le programme d'entrée se situe dans le programme main_receive.py
Il y a deux possibilités : 
   - Soit une exécution directe de ce programme via le terminal (python main_receive.py) 
   - Aussi, si besoin d'exécution à partir d'un autre programme, on appelle une fonction main() qui existe dans ce programme
Les deux possibilités ont le même effet. 
Il faut juste garder à l'esprit que c'est un programme qui introduit une boucle infinie, 
donc si exécution à partir d'un autre programme il faut prendre ça en compte -> Programmation threads, ... 

Que fait ce programme en gros? 

Quand on exécute main() dans main_receive ou bien le programme main_receive directement, voici ce qui se passe :
   - On charge les configurations de la MQTT (on charge le host, port, topic d'écoute) à partir de mqtt_receive_config.yaml qui se situe dans /config
   - Cela s'effectue avec les codes dans config_loader
   - Cela s'effectue une seule fois au début du programme, donc si jamais les cnfigs changent il faut reboot (redémarrer le programme à nouveau)
   - Ensuite, on démarre la boucle infinie pour écouter le topic liées aux données envoyés par le routeur
   - Une fois on reçoit un message (d'après la fiche technique du routeur, celui ci est en JSON) 
   - Le programme appelle writer.py
   - Dans writer, il y a deux fonction, une qui décode le JSON et fournit un tuple des données, et une autre qui écrit le tuple dans la BDD
   - Il s'agit d'une table router_data dans optimasol.db dans database. 
   - Le programme ajoute la date / heure de la réception des données dans la première colonne
   - Comme ça, chaque ligne contient la date, avec 28 autres données liée au routeur.
   - Bien sûr, la table et la BDD, si elles n'existent pas, le programme les crée automatiquement
   - Aussi, si jamais il y a une erreur, le programme affiche les déatails des erreurs, grâce aux print 
   - Ces print doivent être remplcés après par des logs plus formels afin de faciliter le débuggage et de stocker les messages. 

Le dossier tests il contient un JSON déduit à partir de la fiche technique, il sert comme un test dans writer.py, pour voir comment ça marche :
   - exécuter le programme writer.py seul
   - Une ligne s'ajoute dans la DB.
   - C'est toujours la même ligne (à part la date qui se modifie selon la date dans laquelle le programme s'exécute)
